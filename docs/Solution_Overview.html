1. Executive Summary

This solution automates the transfer of employee attendance data from Petra Energy's existing EntryPass access control system into MiHCM's cloud-based HR platform. What was previously a manual, error-prone process requiring daily human intervention is now a fully automated, zero-touch pipeline. The application runs continuously on a Windows workstation at each site, syncing attendance records to MiHCM every 15 minutes with no IT involvement required under normal operation.

Version 1.1.2 introduces a purpose-built Windows GUI application — EntryPassSync — that connects directly to the EntryPass Firebird database, eliminating the dependency on file exports entirely. The application features automatic startup, a built-in sync timer, live status monitoring, and self-updating capability via GitHub.

2. The Challenge

Petra Energy operates EntryPass card readers at multiple sites across Sarawak and Labuan for employee clock-in/clock-out. This attendance data must be fed into MiHCM for payroll processing, leave management, and HR reporting.

Without this integration:

  • HR staff must manually export data from EntryPass at each site daily
  • Exported data must be manually reformatted to match MiHCM's API requirements
  • Records must be manually uploaded to MiHCM via the web portal
  • Formatting errors cause failed imports and payroll delays
  • No audit trail — if something goes wrong, there is no log to trace
  • This manual burden multiplies with every additional site

3. The Solution

EntryPassSync is a purpose-built Windows application developed by Dajayana Trading that bridges EntryPass and MiHCM, handling the complete data pipeline automatically and continuously.

3.1 Architecture Overview

The application runs as a GUI application with a system tray presence on a designated Windows workstation at each Petra Energy site. It reads attendance data directly from the EntryPass Firebird database (the live database used by EntryPass itself) and uploads records to MiHCM's cloud API every 15 minutes. No file exports, no manual steps, no Task Scheduler — the application manages its own schedule.

3.2 Core Capabilities

# Capability Description
1 Database Direct Mode Connects directly to the EntryPass Firebird database (TRANS.FDB) using the .NET Firebird driver. Reads daily attendance tables (STT<YYYYMMDD>) without requiring any file exports from EntryPass
2 Continuous Auto-Sync Syncs immediately on application startup, then automatically every 15 minutes. A live countdown timer on the dashboard shows time until next sync. No Task Scheduler or cron jobs required
3 Auto-Start with Windows Registers itself in the Windows registry (HKLM\Software\Microsoft\Windows\CurrentVersion\Run) on first run as Administrator. Starts automatically when any user logs into Windows — machine-wide, not user-specific
4 Windows GUI & System Tray Full Windows Forms GUI with a Dashboard, Configuration panel, Live Log viewer, and About panel. Minimises to the system tray (with Dajayana logo icon) when closed — keeps running in the background without cluttering the taskbar
5 Multi-Day Catch-Up Configurable sync window (default: 2 days — today and yesterday). Ensures attendance data is not missed over weekends, public holidays, or if the workstation was offline
6 Secure Authentication Authenticates with MiHCM's cloud API using OAuth 2.0 Client Credentials flow. Tokens are managed automatically with mid-session refresh on HTTP 401
7 Batch Upload Uploads records in optimised batches of 80 records per request to MiHCM's attendance endpoint, respecting API payload constraints and rate limits
8 Duplicate Prevention Server-side duplicate detection prevents double-counting in MiHCM. Safe to re-run at any time — already-uploaded records are silently skipped
9 Error Recovery Automatic retry with exponential backoff (2s → 4s → 8s) on network failures, API throttling (HTTP 429), or server errors (5xx). Token refresh on 401. The application recovers without human intervention
10 Location Tagging Each site's data is tagged with its location code (pre-configured per site) for site-level reporting and analytics in MiHCM
11 Live Log Viewer Real-time sync log displayed within the application — IT staff can see exactly what was uploaded, any errors, and when the last sync ran
12 Auto-Update Checks GitHub for new versions on startup. The About panel includes a "Check for Updates" button. Updates are applied by replacing the application file — no reinstallation required
13 License Validation License key validated against GitHub on startup with a local cache fallback. If online validation fails, the cached license permits continued operation — no service interruption during internet outages

4. Security & Connectivity

4.1 Network Requirements

The application requires no inbound network access and opens no listening ports. All communication is outbound HTTPS only:

  • api.mihcm.com — Attendance data upload and OAuth2 token requests
  • raw.githubusercontent.com — License validation and version/update checks

Standard outbound HTTPS (port 443) through the site's existing internet connection is sufficient. No VPN, port forwarding, or firewall rule changes are required beyond allowing outbound HTTPS.

4.2 Credential Management

All sensitive credentials — including the MiHCM API client ID and secret, subscription key, Firebird database credentials, license key, and location code — are stored in a pre-configured config.json file provided by Dajayana Trading. These values are not exposed in the application UI. Site IT staff cannot view or modify API credentials through the interface.

4.3 Data Privacy

Attendance records (card number, timestamp, location) are transmitted directly to MiHCM's authenticated API endpoint over TLS-encrypted HTTPS. No attendance data is stored outside the local machine or transmitted to any third party.

7. Deployment Model

Each site receives a pre-configured deployment package prepared by Dajayana Trading. The package is self-contained and requires no installation wizard, no additional software beyond a running Firebird service, and no configuration by site IT staff.

7.1 Deployment Package (per site)

EntryPassSync.ps1 The main application — PowerShell WinForms GUI. Do not edit.
EntryPassSync.vbs The launcher. Double-click this to start the application. Handles execution policy bypass and hides the PowerShell console.
config.json Pre-configured by Dajayana Trading. Contains license key, MiHCM API credentials, Firebird DB path, location code, and sync settings. Not visible in the UI.
dajayana.ico Application icon displayed in the system tray and taskbar.

7.2 Deployment Steps (site IT)

  • Copy the pre-configured folder to the designated workstation (the same machine running EntryPass or with network access to the Firebird database)
  • Right-click EntryPassSync.vbs and select Run as Administrator for the first launch — this registers the auto-start registry key
  • The application starts, syncs immediately, and begins the 15-minute timer
  • Subsequent launches (after Windows restart) are fully automatic — no user action required
